4. Add the `updateHistory` method after the `update` method:
```typescript
updateHistory(history: DailyAggregate[]): void {
  if (!history || history.length === 0) {
    this.historyBar.style.display = 'none';
    return;
  }

  const totalTokens = history.reduce((sum, d) =>
    sum + d.inputTokens + d.outputTokens + d.cacheCreationTokens + d.cacheReadTokens, 0);
  const totalCost = history.reduce((sum, d) => sum + d.totalCostUsd, 0);
  const totalSavings = history.reduce((sum, d) => sum + d.cacheSavingsUsd, 0);
  const totalSessions = history.reduce((sum, d) => sum + d.sessionCount, 0);
  const dayCount = history.length;

  this.historyBar.style.display = 'flex';
  this.historyBar.innerHTML =
    `<span class="history-label">30-Day:</span>` +
    `<span class="stat"><span class="stat-label">Tokens:</span> ${formatTokens(totalTokens)}</span>` +
    `<span class="stat"><span class="stat-label">Cost:</span> ~$${totalCost.toFixed(2)}</span>` +
    `<span class="stat"><span class="stat-label">Saved:</span> ~$${totalSavings.toFixed(2)}</span>` +
    `<span class="stat"><span class="stat-label">Sessions:</span> ${totalSessions}</span>` +
    `<span class="stat"><span class="stat-label">Days:</span> ${dayCount}</span>`;
}
```

**Design rationale:**
- The history summary uses a muted green-gold color scheme (`#556644`, `#8a8a6e`, `#666644`) to visually distinguish it from today's brighter gold totals (`#c9a96e`, `#887744`). This signals "historical/archival" vs "live/current" at a glance.
- The `30-Day:` label at the start makes it immediately clear what time range the numbers cover.
- Smaller font sizes (12px stats, 10px labels vs 13px/11px for today) reinforce the secondary nature of historical data.
- Hidden by default (`display: none`) -- only shown when actual history data exists. No "loading" or "no history" placeholder.

  window.agentWorld.onDashboardUpdate((data) => {
    dashboardPanel.update(data);

  window.agentWorld.getHistory().then((history) => {

**IMPORTANT constraints:**
- Use `.then()` / `.catch()` instead of `await` because the dashboard panel does not need to wait for history to render. The live dashboard should appear immediately; history loads in the background.
- The history is loaded ONCE at startup. It is not refreshed on every poll cycle. The 30-day summary changes at most once per day (on day rollover), and the user can restart the app to see the updated history.
- Do NOT add a periodic refresh timer for history. This is deferred to a future polish phase if needed.
