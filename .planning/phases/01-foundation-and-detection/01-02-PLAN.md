Create the filesystem-based session detection system:

**src/main/jsonl-reader.ts** -- Efficient tail reader for large JSONL files:

Implement `readLastJsonlLine(filePath: string, bufferSize?: number): { type: string; sessionId?: string; cwd?: string; timestamp?: string; data?: any } | null`

Algorithm:
1. Open file with `fs.openSync(filePath, 'r')`
2. Get file size via `fs.fstatSync(fd)`
3. If size is 0, return null
4. Allocate buffer of `Math.min(bufferSize || 4096, stat.size)` bytes
5. Seek to `stat.size - readSize` and read that chunk
6. Split by newlines, filter empty lines
7. Try `JSON.parse()` on the last non-empty line
8. If parse fails (race condition with Claude writing), try second-to-last line
9. If both fail, return null (don't crash -- log warning)
10. Always close the file descriptor in a finally block

CRITICAL: Do NOT use `fs.readFileSync()` on these files. They can be 2-18 MB. Only read the last 4KB.

**src/main/session-detector.ts** -- Abstract interface + filesystem implementation:

Define the `SessionDetector` interface (per user decision -- abstract for future format changes):
```typescript
export interface SessionDetector {
  discoverSessions(): SessionInfo[];
}
```

Implement `FilesystemSessionDetector`:

1. `discoverSessions()`:
   - Read `~/.claude/projects/` directory (use `os.homedir()` + `path.join`)
   - For each subdirectory, list files matching UUID pattern: `/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}\.jsonl$/`
   - SKIP subdirectories within project dirs (these contain subagent files -- out of scope per research)
   - For each matching JSONL file:
     a. `fs.statSync()` to get mtime
     b. `readLastJsonlLine()` to get last entry
     c. Extract `cwd` from the last entry for project path. Cache the cwd mapping per sessionId so we don't re-read completed sessions unnecessarily
     d. `path.basename(cwd)` for display name
     e. Call `determineStatus()` for status

2. `determineStatus(lastEntryType: string, mtimeMs: number, now: number)`:
   - If `now - mtimeMs > IDLE_THRESHOLD_MS` (30s): return `'idle'`
   - Otherwise switch on lastEntryType:
     - `'assistant'`: if `now - mtimeMs < 2000` return `'active'` else `'waiting'`
     - `'user'` | `'progress'` | `'queue-operation'`: return `'active'`
     - `'system'`: if `now - mtimeMs < 5000` return `'active'` else `'idle'`
     - default: return `'active'` (optimistic per user decision)

3. Error handling:
   - Wrap all filesystem operations in try/catch
   - If a directory or file disappears between readdir and access, skip silently
   - If JSONL can't be parsed and no cached cwd exists, still create a SessionInfo using the directory name as a fallback (not ideal but better than missing the session)
   - Never throw from discoverSessions -- always return a (possibly empty) array

The `cwd` cache is important: once we know a session's project path from reading its JSONL, cache it by sessionId. On subsequent polls, only re-read the last line for sessions with changed mtime. For sessions with unchanged mtime, reuse the cached SessionInfo.

Create the SessionStore that orchestrates detection and pushes updates to the renderer:

**src/main/session-store.ts** -- Central state management:

```typescript
export class SessionStore {
  private sessions: Map<string, SessionInfo>;
  private detector: SessionDetector;
  private pollInterval: NodeJS.Timeout | null;
  private mainWindow: BrowserWindow | null;

  constructor(detector: SessionDetector);
  start(mainWindow: BrowserWindow): void;   // Begin polling
  stop(): void;                              // Stop polling, cleanup
  getSessions(): SessionInfo[];              // Current snapshot
}
```

Implementation details:

1. `start(mainWindow)`:
   - Store mainWindow reference for IPC push
   - Run an immediate first poll (don't wait for interval)
   - Start `setInterval` at `POLL_INTERVAL_MS` (3000ms from constants)
   - Each interval calls `poll()`

2. `poll()`:
   - Call `detector.discoverSessions()` to get fresh SessionInfo[]
   - Compare with previous sessions map to detect changes (new sessions, status changes, removed sessions)
   - Per user decision: completed/ended sessions stay visible until app restart. So NEVER remove a session from the map. If a session was previously known but not returned by the detector (JSONL file deleted?), keep it in the map with its last known state
   - If any changes detected, push update to renderer: `mainWindow.webContents.send(IPC_CHANNELS.SESSIONS_UPDATE, Array.from(sessions.values()))`
   - If no changes, don't send (avoid unnecessary IPC traffic and re-renders)

3. `stop()`:
   - Clear the interval timer via `clearInterval`
   - Set pollInterval to null
   - Set mainWindow to null

4. Change detection:
   - For each session from detector: check if sessionId exists in map and if status or lastModified changed
   - New sessionId = new session
   - Status change = update
   - Any change triggers a push
