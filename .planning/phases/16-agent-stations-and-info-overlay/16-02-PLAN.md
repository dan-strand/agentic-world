  <name>Task 1: Agent interior behavior with wander, scale, z-ordering, and station management</name>
  <files>src/renderer/agent.ts, src/renderer/building.ts, src/renderer/world.ts</files>
  <action>
This task implements the core agent-inside-building behavior: agents walk into buildings, position at stations, wander around their station area, and are properly z-ordered within the building interior.

**A. Agent.ts -- Add wander behavior and interior mode:**

1. Add interior mode properties:
   ```typescript
   // Interior mode (when working inside a building)
   private interiorMode = false;
   private wanderCenterX = 0;
   private wanderCenterY = 0;
   private wanderTimer = 0;
   private wanderTargetX = 0;
   private wanderTargetY = 0;
   ```

2. Add `setInteriorMode(enabled: boolean)` method:
   - When enabled: set `this.scale.set(AGENT_INTERIOR_SCALE)` (1.5x) and `this.interiorMode = true`
   - When disabled: set `this.scale.set(1)` and `this.interiorMode = false`

3. Modify the `working` state in `tick()` to include wander behavior:
   - When in interiorMode and state is 'working':
     - Increment wanderTimer by deltaMs
     - When wanderTimer >= AGENT_WANDER_INTERVAL_MS: pick new random wander target within AGENT_WANDER_RADIUS of wanderCenter, reset timer
     - Move toward wanderTarget at AGENT_INTERIOR_WALK_SPEED
     - Use 'walk' animation while moving to wander target, 'work' animation when arrived (within 2px)
   - Set wanderCenter when entering working state (from walking_to_workspot arrival)

4. Update `assignToCompound()`: no changes needed (already walks to entrance then work spot). The continuous walk from campfire through entrance to station is already handled by the walking_to_building -> walking_to_workspot state transitions.

5. Update `assignToHQ()`: call `setInteriorMode(false)` when leaving a building.

6. When transitioning from walking_to_workspot to working state (hasArrived check):
   - Set `this.wanderCenterX = this.workSpotTarget.x` and `wanderCenterY = this.workSpotTarget.y`
   - Set initial wanderTarget to wanderCenter
   - Reset wanderTimer to 0

**B. Building.ts -- Add station tracking and agent container for z-ordering:**

1. Add station occupancy tracking:
   ```typescript
   private stationOccupancy: Map<number, string> = new Map();  // spotIndex -> sessionId
   ```

2. Add `assignStation(sessionId: string): number` method:
   - Find first unoccupied station (not in stationOccupancy values)
   - If all 3 occupied, pick a random one (4th+ agents share per user decision)
   - Store in stationOccupancy, return the spot index

3. Add `releaseStation(sessionId: string): void` method:
   - Remove sessionId from stationOccupancy

4. Add `reassignStation(sessionId: string): number` method:
   - Release current station, then assign a new random one (different from previous if possible)
   - Used when tool changes trigger station switch

5. Add an `agentsLayer` Container as a child of Building, positioned between floor art (the building sprite) and any elements that should be in front of agents. Since the building is a single sprite, the z-ordering approach is:
   - The Building's children order is: sprite (index 0), agentsLayer (index 1), labelText (index 2), work spot indicators (index 3+)
   - Actually, re-order: sprite first, then agentsLayer, then label and props on top
   - This means agents render on top of the building floor but behind the label text (which acts as a signpost above)
   - For the "behind tall furniture" depth effect: since buildings are a single flat sprite (not layered furniture), agents rendering on top of the sprite IS the correct visual -- they appear "in" the room

6. Add `getAgentsLayer(): Container` getter so World can parent agents into the building's agent layer.

**C. World.ts -- Interior routing and station assignment:**

1. When an agent is assigned to a building (transitions from idle_at_hq to walking):
   - Call `building.assignStation(sessionId)` to get station index
   - Store station index in `agentSpotIndex`
   - Call `agent.setInteriorMode(true)`
   - **Reparent agent** from `this.agentsContainer` to `building.getAgentsLayer()`. This is critical for z-ordering -- agents inside buildings must be children of the building container, not the global agents container.
   - Compute work position in GLOBAL coordinates for the walk target (building.x + local spot position)

2. When agent leaves building (assignToHQ / celebration walkback):
   - Call `building.releaseStation(sessionId)`
   - Call `agent.setInteriorMode(false)`
   - **Reparent agent** back to `this.agentsContainer`

3. When tool changes (activity change while working):
   - Call `building.reassignStation(sessionId)` to get new station
   - Update `agentSpotIndex`
   - Call `agent.updateActivity(newWorkPos)` (already exists)

4. Important coordinate conversion note: When agents are children of the building container, their position is in building-local coordinates. When children of agentsContainer, position is in global/stage coordinates. The reparenting must adjust the agent's position:
   - Moving agent INTO building: `agent.x = agent.x - building.x; agent.y = agent.y - building.y;` (convert global to local)
   - Moving agent OUT of building: `agent.x = agent.x + building.x; agent.y = agent.y + building.y;` (convert local to global)
   - Also update workSpotTarget and buildingEntrance to be in the correct coordinate space

5. Update `getBuildingWorkPosition()` to return LOCAL coordinates (not global) when the agent is inside the building container. Actually, since the agent will be a child of the building, work positions should be in building-local space. The BUILDING_WORK_SPOTS are already in local space, so use them directly without adding building.x/y.

6. Update `getBuildingEntrance()` similarly -- the entrance in local coords is `building.getEntrancePosition()` without adding building.x/y when the agent will be reparented.

7. Actually, the simplest approach: keep using global coordinates for the walk FROM campfire TO entrance (agent is still in agentsContainer during walking_to_building). Then at the transition point (arrival at entrance), reparent into building and convert coordinates. The walking_to_workspot and working states happen with the agent in building-local coords.

8. Implementation detail for reparenting: In the `walking_to_building` -> `walking_to_workspot` transition in Agent.tick(), the agent arrives at the entrance. At this point, World needs to reparent. The cleanest approach: add a callback or flag system. Alternatively, check in World.tick() if any agent just transitioned to walking_to_workspot and needs reparenting.

   **Recommended approach:** Handle reparenting in World's tick loop. After `agent.tick(deltaMs)`, check if agent state just became 'walking_to_workspot' and agent is still in agentsContainer -- if so, reparent into building. Similarly, if agent is leaving building (state becomes walking_to_building with null workSpotTarget = going to HQ), reparent back.

   Add a tracking set `private agentsInBuildings: Set<string> = new Set()` to track which agents have been reparented. In tick:
   - If agent state is 'walking_to_workspot' or 'working' and NOT in agentsInBuildings: reparent into building, add to set
   - If agent state is 'walking_to_building' (going to HQ) or 'idle_at_hq' or 'celebrating' and IS in agentsInBuildings: reparent out, remove from set

9. Clean up `agentsInBuildings` in `removeAgent()`.
