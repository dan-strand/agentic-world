c. **Add the checkAllSessionsWaiting() private method** (near checkForCompletion, around line 600):
   ```typescript
   /**
    * Check if ALL non-idle sessions have committed to "waiting" status.
    * If so, play the global "jobs done" sound exactly once.
    *
    * Requires at least one session to exist (no sound for empty world).
    * Idle sessions are excluded -- they're not "working" so don't count.
    * The jobsDoneFired flag prevents re-triggering until reset by an active commit.
    */
   private checkAllSessionsWaiting(): void {
     let hasNonIdle = false;
     for (const [, status] of this.lastCommittedStatus) {
       if (status === 'idle') continue;
       hasNonIdle = true;
       if (status !== 'waiting') return; // At least one session still active/error
     }
     if (!hasNonIdle) return; // No non-idle sessions, nothing to signal
     // All non-idle sessions are waiting -- play global signal
     this.jobsDoneFired = true;
     SoundManager.getInstance().play();
   }
   ```

**Key design points:**
- `jobsDoneFired` resets on ANY session committing to `active`, not just the one that triggered the original sound (AUDIO-03)
- The check runs every tick but short-circuits immediately via the `!this.jobsDoneFired` guard (negligible cost)
- Idle sessions don't count -- a mix of waiting + idle still triggers the all-clear
- Must have at least one non-idle session (prevents sound on empty world or all-idle)
- Individual celebrations (`agent.startCelebration()`) continue working independently -- only the sound is globalized
