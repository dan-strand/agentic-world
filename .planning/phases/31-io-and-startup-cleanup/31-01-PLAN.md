  <action>
    Restructure `src/main/index.ts` to move all synchronous file I/O out of the module-level scope and into the `app.on('ready', ...)` callback.

    Current module-level code that does sync I/O (to be moved):
    ```
    const historyStore = new HistoryStore(app.getPath('userData'));   // readFileSync in constructor
    const store = new SessionStore(detector, usageAggregator, historyStore);
    const crashLogger = new CrashLogger(app.getPath('userData'));
    crashLogger.checkPreviousCrash();                                // existsSync + readFileSync
    ```

    New structure:
    1. Keep at module level (no I/O): `detector`, `usageAggregator` (these constructors are pure -- no file reads).
    2. Declare `let historyStore: HistoryStore`, `let store: SessionStore`, `let crashLogger: CrashLogger` at module level (uninitialized).
    3. Move into the `app.on('ready', ...)` callback, BEFORE `createWindow()`:
       ```typescript
       app.on('ready', () => {
         historyStore = new HistoryStore(app.getPath('userData'));
         store = new SessionStore(detector, usageAggregator, historyStore);
         crashLogger = new CrashLogger(app.getPath('userData'));
         crashLogger.checkPreviousCrash();
         registerIpcHandlers(store, historyStore);
         createWindow();
       });
       ```
    4. Move `registerIpcHandlers(store, historyStore)` call into the ready callback too (it needs the initialized store/historyStore).
    5. Move IPC listeners that reference `crashLogger` (`CRASH_LOG_ERROR`, `CRASH_LOG_CRITICAL`, `CRASH_MEMORY_STATS`, `CRASH_MEMORY_WARNING`) into the ready callback as well, since crashLogger isn't constructed until then.
    6. Move `process.on('uncaughtException', ...)` and `process.on('unhandledRejection', ...)` into the ready callback since they reference crashLogger. Add a fallback at module level that just does `console.error` + `app.exit(1)` for pre-ready crashes.
    7. Keep `app.on('before-quit', ...)` but guard `historyStore?.flush()` and `store?.stop()` with optional chaining since they may be undefined if quit fires before ready.
    8. Keep `app.on('window-all-closed', ...)` at module level (no deps).
    9. Window control IPC handlers (`window-minimize`, `window-close`, `window-drag-start`, `window-drag-end`) can stay at module level -- they don't reference any of the deferred objects.
