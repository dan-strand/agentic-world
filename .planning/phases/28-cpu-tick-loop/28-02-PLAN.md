2. **Add** reparenting calls in `manageAgents()` at the state transition points where container changes actually happen:

   a. When agent starts walking to workspot (agent arrives at entrance and transitions to walking_to_workspot). This is handled by `agent.assignToCompound()` which sets state to `walking_to_building`. The reparenting INTO a building should happen when the agent arrives at the entrance and state becomes `walking_to_workspot`. Since `handleAgentReparenting` already checks state, we can call it right after `assignToCompound`:
   After line ~668 (`agent.assignToCompound(entrance, workPos);`):
   No change needed here — the agent is still walking_to_building at this point. The actual transition to walking_to_workspot happens in agent.tick() when hasArrived returns true. So we need to keep a lightweight check.

   **REVISED APPROACH**: Instead of removing handleAgentReparenting entirely from tick(), make it O(1) by only calling it for agents whose state JUST changed. Add a `lastTickState` tracking Map:

   ```typescript
   // Track last-tick state per agent for reparenting change detection
   private lastTickState: Map<string, string> = new Map();
   ```

   In tick(), replace the unconditional `this.handleAgentReparenting(agent)` with:
   ```typescript
   // Reparent only when agent state actually changes (not every frame)
   const currentState = agent.getState();
   const prevState = this.lastTickState.get(agent.sessionId);
   if (currentState !== prevState) {
     this.handleAgentReparenting(agent);
     this.lastTickState.set(agent.sessionId, currentState);
   }
   ```

   Also add `this.lastTickState.delete(sessionId)` in `removeAgent()` alongside the other Map deletions, and in the cleanup loop at the end of `manageAgents()`.
