**REWRITE `src/renderer/world.ts`** -- This is a major rewrite, not a minor edit. The new World class manages:

1. **Scene hierarchy:**
   ```
   app.stage
   ├── backgroundContainer (ground, roads)
   ├── hq (HQ Container)
   ├── compoundsContainer (dynamic Compound children)
   └── agentsContainer (dynamic Agent children)
   ```

2. **State tracking:**
   ```typescript
   private agents: Map<string, Agent> = new Map();        // sessionId -> Agent
   private compounds: Map<string, Compound> = new Map();  // projectName -> Compound
   private agentFactory: AgentFactory;
   private hq: HQ;
   ```

3. **`async init(container: HTMLElement)`:**
   - Create PixiJS Application (same config as Phase 1: resizeTo window, BACKGROUND_COLOR, antialias false, resolution devicePixelRatio)
   - Create scene containers in correct z-order
   - Create HQ at center of screen
   - Draw ground/roads background (simplified from Phase 1 -- now just a ground fill + radial road spokes that update dynamically)

4. **`updateSessions(sessions: SessionInfo[])`** -- Core logic, called on every IPC update:

   a. **Group sessions by projectName:**
   ```typescript
   const projectSessions = new Map<string, SessionInfo[]>();
   for (const s of sessions) {
     const list = projectSessions.get(s.projectName) ?? [];
     list.push(s);
     projectSessions.set(s.projectName, list);
   }
   ```

   b. **Manage compounds** -- Spawn/despawn based on active projects:
   - For each projectName with sessions where at least one is NOT idle: ensure a Compound exists
   - For projectNames where ALL sessions are idle (or project disappeared): remove Compound (fade out over 500ms via alpha tween, then remove from container)
   - When compound count changes: recalculate positions using `calculateCompoundPositions()`, animate existing compounds to new positions (simple lerp over 300ms)
   - Draw/update road spokes from HQ center to each compound entrance

   c. **Manage agents** -- Create/update/transition:
   - For each session: ensure an Agent exists (create with AgentFactory slot if new)
   - If session's project has a compound and agent is at HQ: trigger `assignToCompound()` with compound entrance + sub-location for the session's activityType
   - If session's activityType changed and agent is already working: trigger `updateActivity()` with new sub-location position
   - If session's project compound was removed (all idle): trigger `assignToHQ()` to send agent back
   - Agents whose sessions are idle and at HQ: reposition using `hq.getIdlePosition()`

   d. **Handle multiple agents per project:** Multiple sessions for the same project all go to the same compound but to potentially different sub-locations based on their individual activityType.

5. **`tick(deltaMs: number)`:**
   - Tick all agents (agent.tick handles state machine + animation)
   - Tick all speech bubbles (via agents -- each agent owns its bubble)

6. **`resize()`:**
   - Recenter HQ
   - Recalculate compound positions
   - Redraw ground/roads
   - Update agent HQ positions

7. **Road drawing:** Simple radial spokes from HQ center to each compound entrance. Draw as 10px wide filled rects at pathColor (~0x4a4a3a). Redraw when compounds change. Use a dedicated `roadsContainer` between background and compounds in z-order.

8. **Compound fade-in/out:** When spawning, set compound alpha=0 and lerp to 1 over 500ms in tick. When despawning, lerp alpha to 0 over 500ms then remove.
