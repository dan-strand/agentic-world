**src/renderer/agent-sprites.ts** -- Pre-built GraphicsContext animation frames.

Per research: use composited approach to avoid combinatorial explosion. Body frames (color-dependent) and accessory frames are separate layers, combined at render time via child Graphics objects on the Agent Container.

Create an `AgentSprites` class (or module-level functions) that:

1. **Body frames** -- For each of the 8 AGENT_COLORS, pre-build GraphicsContext arrays for these animation states:
   - `idle`: 4 frames with subtle vertical bobbing (2px sinusoidal offset per frame). Character: circle head (skin tone ~0xffd9b3), trenchcoat body (agent color), dark legs (~0x333344).
   - `walking`: 4 frames with alternating leg positions to simulate walking. Slight body lean in movement direction.
   - `working`: 4 frames with arm movement (typing at workbench, adjusting equipment). Character leans forward slightly.

   Each body is drawn at 48x48 scale using Graphics primitives (rect, circle). Do NOT draw individual pixels -- use shape primitives to approximate pixel art:
   - Head: circle at ~(24, 12), radius ~6, skin tone
   - Hat/hair area: rect at top, agent color
   - Body/trenchcoat: rect ~(16, 18, 16, 20), agent color
   - Legs: two rects ~4px wide, dark color, with position varying per walking frame
   - Arms: rects alongside body, position varying per working frame

   Total body contexts: 8 colors * 3 states * 4 frames = 96 GraphicsContext objects.

2. **Accessory frames** -- For each of the 8 ACCESSORIES, pre-build a single GraphicsContext (accessories don't animate independently, they follow the body offset):
   - sunglasses: small dark rect across eye area
   - briefcase: small rect at hand level, brown tone
   - hat: wider brim rect above head (distinct from default hat)
   - scarf: wavy rect at neck level, matches agent color but lighter
   - goggles: two small circles at eye area, strap line
   - earpiece: small dot near ear area
   - badge: small rect on chest area, metallic color
   - tie: thin vertical rect from neck down, contrasting color

   Total accessory contexts: 8 objects.

3. Export an `initAgentSprites()` function that builds all contexts at startup and stores them in a module-level cache. Export getter functions:
   - `getBodyFrames(colorIndex: number, state: 'idle' | 'walking' | 'working'): GraphicsContext[]`
   - `getAccessoryContext(accessory: AccessoryType): GraphicsContext`

**src/renderer/agent-factory.ts** -- Maps sessionId to visual identity.

```typescript
import { AgentSlot } from '../shared/types';
import { getAgentSlot } from '../shared/constants';

export class AgentFactory {
  // Cache slot assignments to avoid rehashing
  private slotCache: Map<string, AgentSlot> = new Map();

  getSlot(sessionId: string): AgentSlot {
    let slot = this.slotCache.get(sessionId);
    if (!slot) {
      slot = getAgentSlot(sessionId);
      this.slotCache.set(sessionId, slot);
    }
    return slot;
  }
}
```

Export a singleton factory instance for use across the renderer.

```typescript
export type AgentState =
  | 'idle_at_hq'
  | 'driving_to_compound'
  | 'walking_to_sublocation'
  | 'working'
  | 'walking_to_entrance'
  | 'driving_to_hq';

export class Agent extends Container {
  readonly sessionId: string;
  private state: AgentState = 'idle_at_hq';
  private bodyGfx: Graphics;         // Swaps context for body animation
  private accessoryGfx: Graphics;    // Swaps context for accessory
  private vehicle: Vehicle;
  private slot: AgentSlot;

  // Movement
  private targetX = 0;
  private targetY = 0;

  // Animation
  private frameIndex = 0;
  private frameTimer = 0;
  private currentBodyFrames: GraphicsContext[] = [];

  // Position references (set by world when assigning to compound/HQ)
  private hqPosition: { x: number; y: number } = { x: 0, y: 0 };
  private compoundEntrance: { x: number; y: number } | null = null;
  private subLocationTarget: { x: number; y: number } | null = null;
}
```

State machine logic in `tick(deltaMs: number)`:
- **idle_at_hq**: Play idle animation at HQ position. Agent visible, vehicle hidden.
- **driving_to_compound**: Move agent+vehicle toward compound entrance at AGENT_DRIVE_SPEED. Play vehicle driving animation. Agent hidden (inside vehicle), vehicle visible. On arrival: transition to `walking_to_sublocation`, hide vehicle (park at entrance), show agent.
- **walking_to_sublocation**: Move agent toward target sub-location at AGENT_WALK_SPEED. Play walking animation. On arrival: transition to `working`.
- **working**: Play working animation at current position. Agent visible, vehicle parked.
- **walking_to_entrance**: Move agent toward compound entrance at AGENT_WALK_SPEED. Play walking animation. On arrival: transition to `driving_to_hq`, hide agent, show vehicle.
- **driving_to_hq**: Move vehicle toward HQ at AGENT_DRIVE_SPEED. On arrival: transition to `idle_at_hq`, hide vehicle, show agent.

Movement uses simple linear interpolation per research Pattern 2:
```typescript
private moveToward(tx: number, ty: number, speed: number, dt: number): void {
  const dx = tx - this.x;
  const dy = ty - this.y;
  const dist = Math.sqrt(dx * dx + dy * dy);
  const step = speed * (dt / 1000);
  if (dist <= step) {
    this.x = tx; this.y = ty;
  } else {
    this.x += (dx / dist) * step;
    this.y += (dy / dist) * step;
  }
}
```

Public methods:
- `constructor(sessionId: string, slot: AgentSlot)` -- creates body/accessory Graphics, vehicle, sets initial frames
- `tick(deltaMs: number): void` -- state machine + animation
- `assignToCompound(entrance: {x,y}, subLocation: {x,y}): void` -- triggers driving_to_compound transition

- `updateActivity(subLocation: {x,y}): void` -- triggers walking to new sub-location within compound
- `setHQPosition(pos: {x,y}): void` -- sets HQ home position
- `getState(): AgentState` -- for external queries
