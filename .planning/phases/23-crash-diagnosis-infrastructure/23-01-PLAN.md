     logError: (source: string, message: string, stack?: string) => void;

3. Create `src/main/crash-logger.ts` with a CrashLogger class:
   - Constructor takes `userDataPath: string` (from `app.getPath('userData')`)
   - Configure electron-log file transport:
     - `resolvePathFn` returns `path.join(userDataPath, 'crash.log')`
     - Format: `[{y}-{m}-{d} {h}:{i}:{s}.{ms}] [{level}] {text}`
     - `maxSize`: 1048576 (1MB rotation)
     - Disable console transport (`log.transports.console.level = false`)
   - Import from `electron-log/main` (NOT plain `electron-log`)
   - Methods:
     - `logCrash(source: string, reason: string, details?: string)`: logs at error level with `[CRASH]` prefix
     - `logError(source: string, error: Error | string)`: logs at error level with `[ERROR]` prefix, includes stack trace for Error objects
     - `logMemoryWarning(message: string)`: logs at warn level with `[MEMORY]` prefix
     - `logMemoryStats(stats: { heapUsedMB: number; rssMB: number })`: logs at info level with `[MEMORY]` prefix, formatted as `heap=X.XMB rss=X.XMB`
     - `logCritical(source: string, message: string)`: logs at error level with `[CRITICAL]` prefix
     - `getLogPath(): string`: returns the log file path
     - `checkPreviousCrash()`: checks if crash.log exists and contains `[CRASH]` or `[CRITICAL]` entries. If yes, logs to stdout via `console.log('[main] Previous crash detected -- see crash.log')`. Uses fs.existsSync + fs.readFileSync, catches errors silently.
   - Export the class

4. Create `src/main/crash-logger.test.ts` using the project's test pattern (node:test, assert/strict, os.tmpdir for temp files):
   - Test that CrashLogger creates the log file in the specified directory
   - Test that logCrash writes an entry containing `[CRASH]`, the source, and the reason
   - Test that logError writes `[ERROR]` with stack trace when given an Error object
   - Test that logMemoryStats writes `[MEMORY]` with formatted heap/rss values
   - Test that logMemoryWarning writes `[MEMORY]` at warn level
   - Test that logCritical writes `[CRITICAL]`
   - Test that checkPreviousCrash detects existing crash entries (create a temp file with a `[CRASH]` line, verify console.log is called)
   - Clean up temp directories in after() hook
   - NOTE: electron-log in test context may need mocking or direct file verification. If electron-log's file transport doesn't work outside Electron, test by verifying the CrashLogger class instantiates correctly and methods don't throw, then verify log file format by reading the file after a short delay (electron-log writes async).   <files>src/main/index.ts, src/preload/preload.ts</files>

1. Modify `src/main/index.ts`:
   - Import CrashLogger: `import { CrashLogger } from './crash-logger';`
   - Create CrashLogger instance at module level (after historyStore): `const crashLogger = new CrashLogger(app.getPath('userData'));`
   - Add `process.on('uncaughtException')` handler EARLY (before app.on('ready')):
     ```
     process.on('uncaughtException', (error) => {
       crashLogger.logCrash('main-uncaughtException', error.message, error.stack);
       // Allow time for electron-log to flush, then exit
       setTimeout(() => app.exit(1), 1000);
     });
     ```
   - Add `process.on('unhandledRejection')` handler:
     ```
     process.on('unhandledRejection', (reason) => {
       const msg = reason instanceof Error ? reason.message : String(reason);
       const stack = reason instanceof Error ? reason.stack : undefined;
       crashLogger.logError('main-unhandledRejection', msg + (stack ? '\n' + stack : ''));
     });
     ```
   - Inside createWindow(), after mainWindow is created, add:
     ```
     mainWindow.webContents.on('render-process-gone', (_event, details) => {
       crashLogger.logCrash('render-process-gone', details.reason, `exitCode=${details.exitCode}`);
     });
     ```
   - Add IPC listeners for renderer crash logging (inside createWindow or at module level after ipcMain imports):
     ```
     ipcMain.on(IPC_CHANNELS.CRASH_LOG_ERROR, (_event, source: string, message: string, stack?: string) => {
       crashLogger.logError(source, stack ? message + '\n' + stack : message);
     });
     ipcMain.on(IPC_CHANNELS.CRASH_LOG_CRITICAL, (_event, source: string, message: string) => {
       crashLogger.logCrash(source, message);
     });
     ipcMain.on(IPC_CHANNELS.CRASH_MEMORY_STATS, (_event, stats: { heapUsedMB: number; rssMB: number }) => {
       crashLogger.logMemoryStats(stats);
     });
     ipcMain.on(IPC_CHANNELS.CRASH_MEMORY_WARNING, (_event, message: string) => {
       crashLogger.logMemoryWarning(message);
     });
     ```
   - Call `crashLogger.checkPreviousCrash()` right after creating the crashLogger instance (before app.on('ready'))
   - In the `before-quit` handler, add a final info log: `crashLogger.logInfo?.('App shutting down normally')` or use the electron-log instance directly if logInfo isn't exposed -- alternatively just keep shutdown simple since electron-log flushes on process exit.

2. Modify `src/preload/preload.ts`:
   - Add `import 'electron-log/preload';` at the top (enables electron-log's IPC bridge in the preload context)
   - Add four new methods to the contextBridge.exposeInMainWorld object:
     ```
     logError: (source: string, message: string, stack?: string): void => {
       ipcRenderer.send(IPC_CHANNELS.CRASH_LOG_ERROR, source, message, stack);
     },
     logCritical: (source: string, message: string): void => {
       ipcRenderer.send(IPC_CHANNELS.CRASH_LOG_CRITICAL, source, message);
     },
     logMemoryStats: (stats: { heapUsedMB: number; rssMB: number }): void => {
       ipcRenderer.send(IPC_CHANNELS.CRASH_MEMORY_STATS, stats);
     },
     logMemoryWarning: (message: string): void => {
       ipcRenderer.send(IPC_CHANNELS.CRASH_MEMORY_WARNING, message);
     },
     ```
