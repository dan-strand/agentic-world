  <name>Task 1: Create MemoryMonitor with sliding window trend detection</name>
  <files>src/renderer/memory-monitor.ts, src/renderer/memory-monitor.test.ts</files>
  <behavior>
    - When sample() is called, it records the heap value and reports stats via the callback
    - When fewer than WINDOW_SIZE samples exist, no warning is triggered
    - When WINDOW_SIZE samples exist and newest - oldest > 50MB, a warning is triggered via callback
    - When WINDOW_SIZE samples exist and newest - oldest <= 50MB, no warning is triggered
    - When more than WINDOW_SIZE samples exist, oldest samples are dropped (sliding window)
    - start() begins periodic sampling; stop() clears the interval
    - stop() is safe to call multiple times
  </behavior>
  <action>
1. Create `src/renderer/memory-monitor.ts`:
   - Constants: `SAMPLE_INTERVAL_MS = 60_000`, `WINDOW_SIZE = 10`, `GROWTH_THRESHOLD_MB = 50`
   - MemoryMonitor class with constructor taking callbacks:
     ```
     interface MemoryMonitorCallbacks {
       onStats: (stats: { heapUsedMB: number; rssMB: number }) => void;
       onWarning: (message: string) => void;
     }
     ```
   - Private fields: `samples: number[]`, `intervalId`, `callbacks`
   - `start()`: sets interval calling `this.sample()`
   - `sample()`: The renderer uses `contextIsolation: true` and `nodeIntegration: false`, so `process.memoryUsage()` is NOT available. Use `performance.memory` (Chromium-specific API) which provides `usedJSHeapSize` and `totalJSHeapSize`. For RSS, use `totalJSHeapSize` as a proxy (true RSS is not available in sandboxed renderer). If `performance.memory` is undefined (non-Chromium), skip sampling and log nothing.
     - Compute heapMB from `performance.memory.usedJSHeapSize / (1024 * 1024)`
     - Push to samples array, shift if > WINDOW_SIZE
     - Call `callbacks.onStats({ heapUsedMB, rssMB })` where rssMB uses `totalJSHeapSize`
     - If samples.length >= WINDOW_SIZE: check `newest - oldest > GROWTH_THRESHOLD_MB` and call `callbacks.onWarning()` with a descriptive message
   - `stop()`: clearInterval, set intervalId to null
   - Export class and constants (export constants for testing)

2. Create `src/renderer/memory-monitor.test.ts` using node:test pattern:
   - Test sliding window: add WINDOW_SIZE samples, verify oldest is dropped on next add
   - Test warning trigger: create monitor, manually call sample() method with mocked performance.memory that returns growing values (>50MB growth), verify onWarning callback fires
   - Test no warning when growth is within threshold
   - Test that onStats callback receives formatted values on each sample
   - Since `performance.memory` doesn't exist in Node.js test environment, the test should construct the MemoryMonitor and call an internal `_sampleWithValues(heapMB, rssMB)` test helper method, OR refactor sample() to accept optional values for testing. The cleanest approach: extract the trend detection logic into a separate pure function `checkTrend(samples: number[], threshold: number): string | null` and test that directly. Export it for testing.

  <name>Task 2: Add game loop error boundary and renderer global error handlers</name>
  <files>src/renderer/game-loop.ts, src/renderer/index.ts</files>
  <action>
1. Modify `src/renderer/game-loop.ts`:
   - Add private fields to GameLoop class:
     ```
     private errorCount = 0;
     private errorWindowStart = 0;
     private stopped = false;
     ```
   - Replace the tickerCallback in start() with a try/catch wrapper:
     ```
     this.tickerCallback = (ticker: { deltaMS: number }) => {
       if (this.stopped) return;
       try {
         this.world.tick(ticker.deltaMS);
         // Reset error count on successful tick
         this.errorCount = 0;
       } catch (err) {
         const now = Date.now();
         if (now - this.errorWindowStart > 10000) {
           // New 10-second window
           this.errorCount = 1;
           this.errorWindowStart = now;
         } else {
           this.errorCount++;
         }

         // Log the error via IPC
         const message = err instanceof Error ? err.message : String(err);
         const stack = err instanceof Error ? err.stack : undefined;
         window.agentWorld.logError('game-loop', message, stack);

         if (this.errorCount >= 5) {
           // Critical: same error repeating rapidly, stop ticker
           window.agentWorld.logCritical('game-loop', 'Ticker stopped: 5+ errors in 10 seconds');
           this.app.ticker.stop();
           this.stopped = true;
         }
       }
     };
     ```

2. Modify `src/renderer/index.ts`:
   - Import MemoryMonitor: `import { MemoryMonitor } from './memory-monitor';`
   - Add global error handlers BEFORE the `main()` call (so they catch init errors too):
     ```
     window.onerror = (message, source, lineno, colno, error) => {
       window.agentWorld.logError(
         'renderer',
         `${message} at ${source}:${lineno}:${colno}`,
         error?.stack
       );
     };

     window.onunhandledrejection = (event) => {
       const reason = event.reason;
       window.agentWorld.logError(
         'renderer-promise',
         reason instanceof Error ? reason.message : String(reason),
         reason instanceof Error ? reason.stack : undefined
       );
     };
     ```

     ```
     const memoryMonitor = new MemoryMonitor({
       onStats: (stats) => window.agentWorld.logMemoryStats(stats),
       onWarning: (message) => window.agentWorld.logMemoryWarning(message),
     });
     memoryMonitor.start();
     ```
