  **1e. Create `assets/sprites/tiles.json` and `assets/sprites/tiles.png` -- Tile atlas:**
  Create the `assets/sprites/` directory. The tile atlas needs at minimum 4 tile types, each 32x32:
  - `grass_1` (primary grass, ~80% of map)
  - `grass_2` (lighter grass variant, ~12%)
  - `grass_3` (darker grass/clover variant, ~8%)
  - `dirt_path` (brown dirt for paths)

  **For the PNG:** Generate a 128x32 pixel PNG programmatically using a Node.js script (run it once, save the output). The script should use the `sharp` library or raw pixel buffer writing to create 4 adjacent 32x32 tiles:
  - Tile 0 (0,0): grass_1 -- medium green base (rgb ~58,120,50) with 2-3 pixel darker spots for texture
  - Tile 1 (32,0): grass_2 -- slightly lighter green (rgb ~72,140,58) with different spot pattern
  - Tile 2 (64,0): grass_3 -- darker green (rgb ~42,100,38) with tiny flower dots (rgb ~180,140,60)
  - Tile 3 (96,0): dirt_path -- brown (rgb ~140,105,60) with lighter grain spots (rgb ~165,130,80)

  Each tile should have subtle pixel variation (not flat color) -- add 3-5 randomly placed single-pixel highlights or shadows within each 32x32 tile to avoid the flat-color look. Use a deterministic approach (fixed coordinates, not Math.random) so the PNG is reproducible.

  Alternative approach if `sharp` is not available: Use a simple script that writes raw RGBA pixel data and pipes through a PNG encoder, OR create the PNG using PixiJS Graphics in a headless Node context, OR manually write the PNG binary. The simplest viable approach: install `pngjs` as a dev dependency (`npm install --save-dev pngjs`) and write the pixels directly.
