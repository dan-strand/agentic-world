  Modify src/renderer/building.ts to store the BitmapText label and expose dynamic label methods:

  1. Add import for MAX_LABEL_CHARS: update the import from '../shared/constants' to include `MAX_LABEL_CHARS` alongside the existing `BUILDING_LABELS`.

  2. Add private properties to the Building class:
     - `private label: BitmapText;` -- stored reference to the BitmapText child
     - `private readonly defaultLabel: string;` -- the RPG name for resetLabel()

  3. In the constructor:
     - Store `this.defaultLabel = BUILDING_LABELS[buildingType];`
     - Change the local `const label` to `this.label` so the BitmapText is stored as a class property. Keep all existing BitmapText config (text, fontFamily, fontSize, anchor, position) identical.

  4. Add setLabel method:
     ```typescript
     setLabel(text: string): void {
       const display = text.length > MAX_LABEL_CHARS
         ? text.slice(0, MAX_LABEL_CHARS - 2) + '..'
         : text;
       if (this.label.text !== display) {
         this.label.text = display;
       }
     }
     ```
     Uses '..' (two ASCII dots) not unicode ellipsis -- stays within the BitmapFont ASCII range. Compares before setting to avoid unnecessary BitmapText layout recalculation.

  5. Add resetLabel method:
     ```typescript
     resetLabel(): void {
       if (this.label.text !== this.defaultLabel) {
         this.label.text = this.defaultLabel;
       }
     }
     ```

  IMPORTANT: Do NOT toggle BitmapText `visible` property -- PixiJS 8 has bug #11294 where text set while invisible does not render correctly when made visible again. Labels stay always visible, only the text content changes.
