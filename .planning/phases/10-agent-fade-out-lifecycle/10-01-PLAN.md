<objective>
Implement the complete agent fade-out lifecycle so completed agents gracefully leave the world instead of accumulating at Guild Hall forever.

Purpose: This is an always-on app. Without cleanup, completed agents pile up at Guild Hall as invisible memory-consuming containers. This phase adds a 6th agent state (fading_out), a single removeAgent() cleanup method that destroys all tracking, and a dismissedSessions guard to prevent stale IPC data from resurrecting removed agents.

Output: Modified agent.ts (6-state FSM), world.ts (removal pipeline + resurrection guard), game-loop.ts (animation-aware FPS), constants.ts (timing), agent-factory.ts (slot release).
</objective>
