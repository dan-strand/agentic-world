## Cross-Phase Integration

### Wired Connections (Phase 11)

All Phase 11 exports are properly consumed:
- `determineStatus()` with `hasToolUse` parameter -> called from both cached and fresh paths in `processSessionFile()`
- `cancelFadeOut()` hardening -> called from `world.ts` manageAgents with debounce reinit
- `lastEntryType` Map -> set in manageAgents, consumed by checkForCompletion, cleaned in removeAgent
- Visibility safeguard -> runs in tick loop, reads agent.alpha and agent.getState()

### Missing Connections (Phases 12-13)

**Phase 12 foundation ready:** `lastCommittedStatus` Map in world.ts contains every agent's committed status. Phase 12 needs to iterate and check if all values are `waiting`. Data is maintained by Phase 11's debounce pipeline.

**Phase 12 must replace:** `SoundManager.play()` at world.ts:209 fires per-session inside `checkForCompletion`. This is the anti-pattern AUDIO-02 prohibits. Phase 12 must replace or gate this call.

**Phase 13 foundation ready:** `lastCommittedStatus` correctly tracks `waiting` status. Phase 13 needs a new `waitingTimers` Map alongside existing `idleTimers`, keyed on `waiting` (not idle).
